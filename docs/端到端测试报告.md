### ❌ 遇到的问题

#### 1. 知识图谱构建超时

**端点**: `POST /api/build_graph/{job_id}?max_chunks=10`

**问题**: 
- LLM调用DeepSeek API处理10个chunks超时（>120秒）
- 每个chunk需要10-15秒（LLM提取知识点+关系）
- 10个chunks总计需要100-150秒

**错误信息**:
```
requests.exceptions.ReadTimeout: HTTPConnectionPool(host='localhost', port=8000): 
Read timed out. (read timeout=120)
```

**根本原因**:
- DeepSeek API响应慢（网络延迟 + 模型推理时间）
- 每个chunk的prompt较长（~1000 tokens）
- 需要生成结构化JSON输出（增加推理时间）

**影响范围**:
- 知识图谱构建失败
- 后续步骤（去重、报告生成、RAG）无法执行

#### 2. 控制台中文乱码

**问题**: Windows GBK编码导致中文显示为乱码

**影响**: 仅影响控制台显示，不影响功能

**示例**:
```
Content: �� �� ѧ Shenglixue ��10 �� ...
```

**实际内容**: 文件中UTF-8编码正常，只是print时乱码
