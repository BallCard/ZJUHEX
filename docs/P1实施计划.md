### Step 4: 异步抽取和缓存 (50分钟)

**目标**: 解决LLM超时和重复调用

**当前问题**: 
- 单次HTTP请求处理10个chunks需100-150秒，超过120秒超时
- 失败后重跑需重新调用LLM

**任务**:
1. 创建`src/backend/services/async_extractor.py`:
   ```python
   class AsyncExtractor:
       def __init__(self, job_id: str):
           self.cache_dir = RUNTIME_DIR / job_id / "extraction_cache"
           self.cache_dir.mkdir(exist_ok=True)
       
       def extract_chunk(self, chunk: Dict) -> Optional[Dict]:
           """
           提取单个chunk的知识点
           1. 检查缓存
           2. 调用LLM
           3. 保存缓存
           4. 返回结果或None（失败）
           """
           cache_file = self.cache_dir / f"{chunk['chunk_id']}.json"
           
           # 检查缓存
           if cache_file.exists():
               return json.load(cache_file.open())
           
           # 调用LLM
           try:
               result = self._call_llm(chunk)
               cache_file.write_text(json.dumps(result, ensure_ascii=False))
               return result
           except Exception as e:
               return {"error": str(e), "chunk_id": chunk["chunk_id"]}
       
       def extract_batch(self, chunks: List[Dict]) -> Dict:
           """
           批量提取，返回进度
           """
           results = []
           for i, chunk in enumerate(chunks):
               result = self.extract_chunk(chunk)
               results.append(result)
               
               # 更新进度
               self.update_progress(i + 1, len(chunks))
           
           return {"results": results, "success_count": len([r for r in results if "error" not in r])}
   ```

2. 修改`main.py`使用后台任务:
   ```python
   from fastapi import BackgroundTasks
   
   @app.post("/api/build_graph/{job_id}")
   async def build_graph(job_id: str, background_tasks: BackgroundTasks):
       # 立即返回，后台处理
       background_tasks.add_task(build_graph_task, job_id)
       return {"status": "processing", "job_id": job_id}
   
   def build_graph_task(job_id: str):
       try:
           extractor = AsyncExtractor(job_id)
           chunks = load_chapter_chunks(job_id)
           result = extractor.extract_batch(chunks)
           
           # 保存结果
           save_graph(job_id, result)
   ```

## P1验收标准

P1完成必须同时满足：

1. ✅ **真实数据**: 处理`03_生理学.pdf`的第一章完整内容
2. ✅ **真实范围**: 报告中的原文范围和整合范围一致
3. ✅ **真实结果**: 整合报告包含知识内容，不只是流程状态
4. ✅ **可追溯**: 每个知识点能追溯到页码和chunk
5. ✅ **可演示**: 前端能完成上传、处理进度、报告查看、图谱查看、RAG问答
6. ✅ **可失败**: API失败时前端不显示成功，state中保留失败原因
7. ✅ **可复跑**: 重复运行同一章节能复用抽取缓存

## 风险红线

出现以下情况时必须停止扩展功能，先修主线：

- ❌ 压缩比达标但分母包含未处理内容
- ❌ 图谱节点主要来自封面、序言、目录
- ❌ API超时后前端仍显示完成
- ❌ LLM失败后系统保存空图谱并继续执行
- ❌ 报告无法说明任何合并/删除理由
- ❌ RAG引用和回答内容明显不对应
