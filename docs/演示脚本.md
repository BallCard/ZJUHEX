## 准备工作（演示前5分钟）

### 1. 启动后端服务

```bash
cd src/backend
uvicorn main:app --reload --port 8000
```

**验证**: 访问 http://localhost:8000/docs 确认API文档可访问

### 2. 打开前端界面

在浏览器中打开: `src/frontend/index.html`

或使用本地服务器:
```bash
cd src/frontend
python -m http.server 3000
```

访问: http://localhost:3000

### 3. 准备测试数据

确认以下文件存在:
- `data/textbooks/03_生理学.pdf`
- `data/textbooks/02_组织学与胚胎学.pdf`

### 4. 清理旧数据（可选）

```bash
# 清理运行时数据
rm -rf data/runtime/jobs/*

# 清理报告
rm -rf report/*
```

## 演示流程

### Part 1: 单教材演示（5分钟）

**目标**: 展示单教材知识整合的完整流程

#### 1.1 上传教材（30秒）

**操作**:
1. 选择"单教材模式"
2. 点击上传区域
3. 选择 `03_生理学.pdf`
4. 等待上传完成

**说明**:
> "系统支持PDF格式的医学教材上传，使用PyMuPDF进行文本提取。"

#### 1.2 运行整合流水线（2分钟）

**操作**:
1. 点击"运行整合流水线"按钮
2. 观察进度条实时更新

**说明**:
> "流水线包含5个阶段：
> 1. 文档解析 - 提取文本并分块
> 2. 知识图谱构建 - LLM提取知识点和关系
> 3. 语义去重 - 使用sentence-transformers计算相似度
> 4. 生成报告 - 计算压缩比和统计信息
> 5. 建立RAG索引 - FAISS向量检索"

**关键指标**:
- 解析块数: ~50-100块
- 知识点数: ~15-30个（前3块）
- 压缩比: 应≤30%

#### 1.3 查看知识图谱（1分钟）

**操作**:
1. 点击"查看知识图谱"按钮
2. 演示交互功能:
   - 缩放图谱（鼠标滚轮）
   - 拖拽节点
   - 点击节点查看详情
   - 点击关联节点导航

**说明**:
> "使用Cytoscape.js实现交互式图谱可视化。节点代表知识点，边代表知识关系（前置依赖、并列、包含、应用）。点击节点可查看定义、来源页码和关联知识点。"

**关键截图**: 图谱可视化界面 + 节点详情面板

#### 1.4 查看整合报告（30秒）

**操作**:
1. 点击"查看报告"按钮
2. 滚动查看报告内容

**说明**:
> "整合报告包含：
> - 压缩比统计（目标≤30%）
> - 知识点列表
> - 去重决策
> - 来源追踪"

**关键指标**: 压缩比≤30%

#### 1.5 RAG问答（1分钟）

**操作**:
1. 在问答框输入: "细胞膜的结构是什么？"
2. 点击"提问"
3. 查看回答和引用来源

**说明**:

**关键截图**: RAG问答结果 + 引用来源

## Q&A准备

### 预期问题

**Q1: 为什么选择单Agent架构而不是多Agent？**

A: 本项目任务流程确定性强（Parse → Extract → Deduplicate → Compress），无需Agent间协商。单Agent架构开发时间2-3小时，适合5小时黑客松约束。多Agent架构（CrewAI/AutoGen）开发时间4-5小时，且存在上下文碎片化风险。详见 `docs/Agent架构说明.md` 第2.1节。

**Q2: 压缩比如何保证≤30%？**

A: 系统采用两层压缩策略：
1. 语义去重：使用0.90相似度阈值合并重复知识点
2. 报告生成：计算（整合后字符数 / 原始字符数）× 100%

如果压缩比>30%，可以调整相似度阈值（降低到0.88-0.89）或增加处理块数。

**Q3: 如何处理7本教材？**

A: 系统支持跨教材整合模式，可同时上传2-7本教材。当前演示使用2本教材（时间约束），但架构支持扩展到7本。处理7本完整教材预计需要15-20分钟（取决于教材大小）。

**Q4: RAG如何保证引用准确性？**

A: RAG流水线包含3个保障机制：
1. FAISS向量检索：召回语义相关内容
2. 引用来源追踪：每个chunk保留教材名、页码、内容
3. Prompt约束：要求LLM只基于上下文回答，并标注引用

未来可引入Judge Model验证引用真实性（PaperQA2-style）。

**Q5: 系统如何部署？**

A: 当前版本（P2）支持本地部署：
- 后端：uvicorn运行FastAPI
- 前端：单HTML文件，直接打开或使用http.server

P3计划支持Docker容器化部署，一键启动。

**Q6: 如何扩展到其他学科？**

A: 系统设计是学科无关的，只需：
1. 替换教材PDF文件
2. 调整Prompt模板（如果需要特定领域术语）
3. 调整相似度阈值（不同学科可能需要不同阈值）

配置化设计使得这些调整无需修改代码。

## 应急预案

### 场景1: 后端服务启动失败

**原因**: 端口占用、依赖缺失

**解决**:
```bash
# 更换端口
uvicorn main:app --reload --port 8001

# 重新安装依赖
pip install -r requirements.txt
```

### 场景2: 图谱无法渲染

**原因**: Cytoscape.js加载失败、数据格式错误

**解决**:
1. 检查浏览器控制台错误
2. 使用备用截图演示
3. 展示JSON格式的图谱数据

### 场景3: LLM API调用失败

**原因**: API密钥无效、网络问题

**解决**:
1. 检查.env中的DEEPSEEK_API_KEY
2. 使用预先运行的结果演示
3. 展示缓存的报告和图谱

### 场景4: 时间不足

**解决**:
1. 跳过单教材演示，直接演示跨教材
2. 使用快速演示流程（5分钟）
3. 重点展示Agent架构说明（20分关键）
