# 跨教材整合功能使用指南

## 快速开始

### 1. 启动后端服务

```bash
cd src/backend
uvicorn main:app --reload --port 8000
```

### 2. 打开前端

在浏览器中打开 `src/frontend/index.html`

### 3. 使用跨教材整合

#### 步骤1: 选择模式
- 在"上传教材"区域，选择"跨教材整合模式"单选按钮

#### 步骤2: 上传多个教材
- 点击上传区域
- 在文件选择对话框中，按住Ctrl键（Windows）或Cmd键（Mac）选择多个PDF文件
- 点击"打开"

#### 步骤3: 运行整合流水线
- 上传成功后，点击"运行跨教材整合流水线"按钮
- 系统会依次执行：
  1. 解析所有教材
  2. 构建知识图谱（可能需要几分钟）
  3. 跨教材整合

#### 步骤4: 查看结果
- 整合统计：查看原始节点数、整合后节点数、去重率
- 查看报告：点击"查看报告"按钮，查看详细的跨教材整合报告
- 知识图谱：在图谱区域查看整合后的知识图谱（如果已实现可视化）

## 跨教材整合报告内容

报告包含以下部分：

### 1. 整合概览
- 教材数量
- 原始知识点总数
- 整合后知识点数
- 去重率
- 跨教材重复知识点数量

### 2. 各教材贡献度
表格展示每本教材的：
- 贡献知识点数
- 独有知识点数（仅在该教材中出现）
- 共享知识点数（与其他教材重复）

### 3. 跨教材重复知识点
列出前10个跨教材重复的知识点，包括：
- 知识点名称
- 出现在哪些教材中
- 定义摘要

### 4. 各教材独有知识点
按教材分类展示独有知识点（前5个）

### 5. 知识互补性分析
- 知识重叠度百分比
- 知识互补度百分比
- 整合价值评估结论

## API使用示例

### 使用curl测试

```bash
# 1. 上传多个教材（需要实际文件路径）
curl -X POST http://localhost:8000/api/upload_multiple \
  -F "files=@data/textbooks/03_生理学.pdf" \
  -F "files=@data/textbooks/02_组织学与胚胎学.pdf"

# 记录返回的job_id，例如: abc12345

# 2. 解析所有教材
curl -X POST http://localhost:8000/api/parse_multiple/abc12345

# 3. 构建知识图谱（异步，立即返回）
curl -X POST http://localhost:8000/api/build_graphs_multiple/abc12345

# 4. 轮询进度（每3秒查询一次）
curl http://localhost:8000/api/jobs/abc12345/progress

# 当status变为"graphs_built"时，继续下一步

# 5. 执行跨教材整合
curl -X POST http://localhost:8000/api/cross_integrate/abc12345

# 6. 获取整合报告
curl http://localhost:8000/api/cross_report/abc12345

# 7. 获取整合图谱（JSON格式）
curl http://localhost:8000/api/cross_graph/abc12345
```

### 使用Python requests

```python
import requests
import time

API_BASE = "http://localhost:8000"

# 1. 上传多个教材
files = [
    ('files', open('data/textbooks/03_生理学.pdf', 'rb')),
    ('files', open('data/textbooks/02_组织学与胚胎学.pdf', 'rb'))
]
response = requests.post(f"{API_BASE}/api/upload_multiple", files=files)
job_id = response.json()['job_id']
print(f"Job ID: {job_id}")

# 2. 解析所有教材
response = requests.post(f"{API_BASE}/api/parse_multiple/{job_id}")
print(f"Parsed: {response.json()['total_chunks']} chunks")

# 3. 构建知识图谱
response = requests.post(f"{API_BASE}/api/build_graphs_multiple/{job_id}")
print("Building graphs...")

# 4. 轮询进度
while True:
    response = requests.get(f"{API_BASE}/api/jobs/{job_id}/progress")
    progress = response.json()
    status = progress['status']
    print(f"Status: {status}")
    
    if status == 'graphs_built':
        break
    elif status == 'failed':
        print(f"Error: {progress.get('error')}")
        exit(1)
    
    time.sleep(3)

# 5. 跨教材整合
response = requests.post(f"{API_BASE}/api/cross_integrate/{job_id}")
result = response.json()
print(f"Integrated: {result['original_nodes']} -> {result['merged_nodes']} nodes")

# 6. 获取报告
response = requests.get(f"{API_BASE}/api/cross_report/{job_id}")
report = response.json()['content']
print("\n=== 跨教材整合报告 ===")
print(report)
```

## 注意事项

### 1. 处理时间
- 单个教材的知识图谱构建需要1-3分钟（取决于章节大小）
- 多个教材会顺序处理，总时间 = 教材数量 × 单个处理时间
- 建议使用进度轮询功能监控处理状态

### 2. 相似度阈值
- 当前使用0.90作为语义相似度阈值
- 该阈值适合生物医学领域，平衡了去重效果和准确性
- 相似度 >= 0.90 的知识点会被合并

### 3. 内存使用
- 每个教材的图谱会单独保存到文件
- 整合时会加载所有图谱到内存
- 建议先测试2-3本教材，确认系统稳定后再处理7本

### 4. 单教材模式兼容性
- 跨教材整合功能不影响原有的单教材模式
- 两种模式可以独立使用
- 单教材模式仍然使用原有的API端点

## 故障排查

### 问题1: 上传失败
- 检查文件格式是否为PDF
- 检查文件大小（建议 < 100MB）
- 检查后端服务是否正常运行

### 问题2: 图谱构建超时
- 这是正常现象，图谱构建是异步的
- 使用进度轮询API检查状态
- 不要重复点击"运行流水线"按钮

### 问题3: 整合结果为空
- 检查是否所有教材都成功构建了图谱
- 查看后端日志确认是否有错误
- 确认教材内容是否包含可提取的知识点

### 问题4: 相似度过低，没有合并
- 这可能是正常的，说明教材内容互补性强
- 可以在报告中查看"知识互补性分析"部分
- 如果需要调整阈值，修改`cross_textbook_integration.py`中的`similarity_threshold`参数

## 性能优化建议

### 开发环境
- 使用2-3本教材测试功能
- 使用`max_chunks`参数限制处理范围（如前10个chunks）

### 生产环境
- 考虑使用多进程并行构建图谱
- 增加服务器内存（建议 >= 8GB）
- 使用Redis缓存中间结果
- 考虑使用GPU加速embedding计算
