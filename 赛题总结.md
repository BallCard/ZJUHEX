# 学科知识整合智能体 - AI全栈黑客松

## 赛题概述

开发一个AI智能体，对7本医学教材进行知识整合：
- 构建可视化知识图谱
- 跨教材去重提纯
- RAG精准问答
- 自主设计Agent架构

**目标**：将7本教材压缩到不超过原始体量30%的精华版本，且教学效果不打折。

**比赛时长**：5小时

## 核心功能要求（P0）

### 1. 多格式教材加载与解析
- 支持PDF（必须）、Markdown（必须）、TXT（必须）、Word .docx（建议）
- 前端文件上传（拖拽+批量）
- 章节结构识别
- 统一输出结构化数据

### 2. 单本教材知识图谱构建
- LLM提取核心知识点（概念、定理、方法、现象）
- 识别知识点关系（前置依赖、并列、包含、应用）
- 可视化展示

### 3. 知识图谱交互
- 节点点击查看详情
- 频次可视化（节点大小/颜色）
- 教材来源区分
- 缩放、拖拽、搜索

### 4. 跨教材知识图谱整合（核心难点）
- 语义对齐（识别不同教材中的同一概念）
- 整合决策（merge/keep/remove）
- 压缩比控制（≤30%）
- 可视化对比

### 5. RAG精准问答
- 文档分块（500-800字，50-100字重叠）
- 向量嵌入（sentence-transformers或OpenAI）
- 向量检索（FAISS/ChromaDB）
- 带引用来源的回答

### 6. Agent架构设计
- 提交`docs/Agent架构说明.md`
- 架构总览、设计决策论证、数据流、取舍权衡

### 7. 整合建议与多轮对话
- 对整合决策提供解释
- 支持教师通过对话修改整合方案
- 对话历史持久化

### 8. Web交互界面
- 单页应用（SPA）
- 左侧：教材管理
- 中间：知识图谱可视化
- 右侧：功能面板（整合/RAG/对话/报告）

### 9. 整合报告
- 整合概览（压缩比）
- 整合决策摘要
- 知识图谱统计
- 重点整合案例
- 教学完整性说明

### 10. 开发文档
- `docs/需求分析.md`
- `docs/系统设计.md`
- `docs/Agent架构说明.md`
- `README.md`

## 加分项（P1）

- 知识图谱多视图切换
- 混合检索（向量+BM25）+ Rerank
- Token消耗统计
- 本地部署开源模型
- Docker一键部署
- 自建RAG Benchmark

## 挑战加分项（P2）

技术报告（3000-8000字），论证技术方案如何提升AI性能，必须有实验数据支撑。

## 评分标准（100分）

- A. 文档完整性与可复现性：15分
- B. 功能实现完整度：25分
- C. 知识图谱可视化创新性：13分
- D. Agent架构设计：20分
- E. 代码质量与工程规范：17分
- F. 创新与自由发挥：10分

## 技术栈建议

- 后端：FastAPI (Python)
- 前端：React / Vue 3
- 知识图谱可视化：D3.js / ECharts / Cytoscape.js / AntV G6
- 大模型：通义千问 / DeepSeek / Claude / OpenAI
- 向量嵌入：sentence-transformers / OpenAI Embedding / BGE
- 向量检索：FAISS / ChromaDB / Qdrant
- Agent框架：LangGraph / CrewAI / AutoGen（可选）
- 文件解析：PyMuPDF / python-docx / pandas
- 部署：魔搭创空间
